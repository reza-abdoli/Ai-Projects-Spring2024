def VGGupdated(input_tensor=None,classes=10):    
   
    img_rows, img_cols = 224, 224 
    img_channels = 3

    img_dim = (img_rows, img_cols, img_channels)
   
    img_input = Input(shape=img_dim)
    
    # Block 1
    x = Conv2D(64, (3, 3), activation='relu', padding='same', name='block1_conv1')(img_input)
    x = Conv2D(64, (3, 3), activation='relu', padding='same', name='block1_conv2')(x)
    x = MaxPooling2D((2, 2), strides=(2, 2), name='block1_pool')(x)

    # Block 2
    x = Conv2D(128, (3, 3), activation='relu', padding='same', name='block2_conv1')(x)
    x = Conv2D(128, (3, 3), activation='relu', padding='same', name='block2_conv2')(x)
    x = MaxPooling2D((2, 2), strides=(2, 2), name='block2_pool')(x)

    # Block 3
    x = Conv2D(256, (3, 3), activation='relu', padding='same', name='block3_conv1')(x)
    x = Conv2D(256, (3, 3), activation='relu', padding='same', name='block3_conv2')(x)
    x = Conv2D(256, (3, 3), activation='relu', padding='same', name='block3_conv3')(x)
    x = MaxPooling2D((2, 2), strides=(2, 2), name='block3_pool')(x)

    # Block 4
    x = Conv2D(512, (3, 3), activation='relu', padding='same', name='block4_conv1')(x)
    x = Conv2D(512, (3, 3), activation='relu', padding='same', name='block4_conv2')(x)
    x = Conv2D(512, (3, 3), activation='relu', padding='same', name='block4_conv3')(x)
    x = MaxPooling2D((2, 2), strides=(2, 2), name='block4_pool')(x)

    # Block 5
    x = Conv2D(512, (3, 3), activation='relu', padding='same', name='block5_conv1')(x)
    x = Conv2D(512, (3, 3), activation='relu', padding='same', name='block5_conv2')(x)
    x = Conv2D(512, (3, 3), activation='relu', padding='same', name='block5_conv3')(x)
    x = MaxPooling2D((2, 2), strides=(2, 2), name='block5_pool')(x)

  
    model = Model(inputs = img_input, outputs = x, name='VGGdemo')


    return model

Q1

  • Extracting features from images using models like VGG16 serves several purposes:
    • Conversion of image information into feature space: Images contain a wealth of information that isn't readily understandable by simply looking at pixels.
    • Transfer learning: Features extracted from pre-trained models like VGG16 can be used as input for new models trained for specific tasks such as object detection in images
    • Deeper semantic understanding: By using features extracted from deep neural network models, we can obtain meaningful and valuable insights from images.

Q2

    1. Convolutional Neural Networks (CNNs): CNNs are powerful tools for extracting features from images. These networks automatically extract features ranging from low-level features like edges and corners to high-level features like objects and meaningful patterns from images.
    2. Wavelet Transform-based Methods: These methods use wavelet analysis to decompose images into components of different frequencies.
    3. Geometric and Statistical Features-based Methods: These methods extract geometric features such as shape, size, and object positions from images.

Q3

  • Preprocessing of images before feeding them into a model typically involves the following steps:
    • Resizing: Images usually come in different sizes, and for optimal use with models, they need to be resized to a specific, often square, size.
    • Normalization: This step involves scaling the pixel values to a specific range, such as [0, 1] or [-1, 1]. Normalization ensures that input values to the model fall within appropriate ranges and addresses issues like saturation or inconsistency in pixel values.
    • Conversion to Model's Input Format: Some models may require images to be in specific input formats, such as CNN models often expecting RGB images with certain dimensions. Therefore, images need to be converted to this format.

import os
def read_and_preprocess_images(directory_path):
    images = []
    for i in range(1, 211):
        filename = '{:04d}.png'.format(i)
        image_path = os.path.join(directory_path, filename)
        if os.path.exists(image_path):
            img = image.load_img(image_path, target_size=(224, 224))
            img_array = image.img_to_array(img)
            img_array = np.expand_dims(img_array, axis=0)
            img_array = preprocess_input(img_array)
            images.append(img_array)
        else:
            print(f"Image file {filename} not found.")
    
    return np.vstack(images)

Q8

  • Homogeneity:
    • To calculate this metric, we first compare the actual data labels and the clusters that the clustering algorithm created.

If a cluster includes only members of one class, the homogeneity criterion of that cluster is 1. But if the data of one class are located in several clusters, the homogeneity criterion will be less than 1. This measure is useful for cases where real labels are available, as we can compare the performance of the clustering algorithm with real labels. are available.

  • Silhouette Score:
    • This score indicates how similar an object is to its own cluster (integration) compared to other clusters (separation). The silhouette score is in the range of -1 to 1, where a higher value means a good match of the object with its own cluster and also a lack of match with the neighboring clusters.
